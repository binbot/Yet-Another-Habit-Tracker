## Remaining Scrolling Judder Issues

### Issue 1: CustomTheme per Habit Item

**Status**: Deferred - required for per-habit colors

**Current State**:
```kotlin
CustomTheme(
    primaryColor = habit.color,  // Different per habit
    isAmoled = allPreferences.getAmoledThemeMode(),
    theme = allPreferences.getTheme()
) {
    HabitItemReorderableNew(...)
}
```

**Why It Causes Issues**:
- Each habit generates a new `rememberDynamicColorScheme` with different primary color
- Material3 components reading `colorScheme` may recompose when any nearby theme changes
- More pronounced in Compose than Flutter due to CompositionLocal behavior

**What Table Habit Does Differently** (Flutter):
- Uses `InheritedWidget` which propagates changes more surgically
- May derive colors at data level rather than theme level

**Previous Attempt** (caused issues):
- An attempt was made to restructure colors but broke the in-app color picker/palette
- The color picker likely depends on `CustomTheme` to preview how a color looks

**Approach for Future Fix**:
1. Do NOT remove CustomTheme entirely
2. Instead, extract ONLY the color derivation logic:
   ```kotlin
   // Create a simple color derivation that CustomTheme can still use
   val habitColors = rememberDynamicColorScheme(
       primary = habit.color,
       isDark = isDarkTheme,
       isAmoled = isAmoled
   )
   // Pass colors as simple Color values, not wrapped in MaterialTheme
   ```
3. Keep CustomTheme for the color picker preview functionality
4. Use direct color assignments in habit items instead of MaterialTheme.colorScheme
